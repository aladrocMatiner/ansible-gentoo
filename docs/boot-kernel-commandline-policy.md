# Boot Kernel Command Line Policy

The installed Gentoo system must boot using a stable root filesystem identifier and the filesystem-specific options generated by the installer. This policy applies to kernel installation, fstab generation, GRUB configuration, final checks, and first-boot validation.

## Root Identification

The generated kernel command line must identify the root filesystem by UUID:

```text
root=UUID=<root-uuid>
```

It must not infer the root disk from a default device path such as `/dev/sda`, `/dev/nvme0n1`, or `/dev/vda`.

## ext4

For `FILESYSTEM=ext4`, the generated command line uses:

```text
root=UUID=<root-uuid> rootfstype=ext4
```

## Btrfs

For `FILESYSTEM=btrfs`, the generated command line uses:

```text
root=UUID=<root-uuid> rootfstype=btrfs rootflags=subvol=@
```

The `rootflags=subvol=@` value must stay aligned with `docs/btrfs-layout-policy.md`. OpenRC and systemd flows must reuse the same Btrfs command line policy.

## v1 Exclusions

The v1 command line must not include:

- LUKS or dm-crypt parameters,
- resume/swap parameters,
- BIOS-only boot assumptions,
- unsupported root filesystem arguments,
- host-specific debug flags unless an approved OpenSpec change adds them.

## Makefile Workflows

The policy is produced and validated through:

```sh
make generate-fstab
make install-kernel
make install-bootloader
make final-checks
make vm-validate-first-boot
make install-report
```

## Validation

Validation must check:

- `/etc/fstab` uses the same root UUID,
- `/mnt/gentoo/etc/kernel/cmdline` contains the approved root command line,
- GRUB config contains the expected root UUID,
- Btrfs boot config contains `rootflags=subvol=@`,
- v1-excluded arguments are absent.

Final checks and first-boot validation must fail closed when root UUID or Btrfs root flags do not match the generated target configuration.

## Failure Modes

- Missing root UUID: rerun `make generate-fstab` after filesystems are mounted and UUIDs are visible.
- Btrfs command line missing `rootflags=subvol=@`: verify the fstab and Btrfs layout policy, then rerun `make install-kernel` and `make install-bootloader`.
- GRUB config lacks `root=UUID=`: rerun `make install-bootloader` only after reviewing the bootloader preview and required confirmation.
- LUKS/resume arguments appear in v1: reject the change unless a later OpenSpec policy adds that feature.

## Recovery

Do not hand-edit boot files as the first response. Rerun the Makefile-mediated fstab, kernel, bootloader, and final-check targets so evidence remains aligned with the install state.
